Rapport

HTML

Des blocs HTML en diplay none trainaient (tests, lorem ipsum, vide...) Je les ai supprimés.

SVG

Je pensais pouvoir faire la mission rapidement, mais j'ai beaucoup peiné.

Les SVGs utilisaient des style css dans les tags, des class pour les différentes <line> SVG, des balises style à l'intérieure des <svg>... c'était un vrai foutoir.

À l'avenir, il ne faudrait pas utiliser un mix de CSS/SVG de cette façon. Si vraiment c'est inévitable, le client devrait essayer de mettre tout le CSS au meme endroit.

Mettre des classes dans les stroke c'est quand on utilise des effets complexes/animations, pas quand on veut juste présenter une image à taille fixe...

CSS

Plusieurs pages utilisaient des kilomètres de CSS n'ayant rien à voir avec la page.

D'autres avaient des doublons, voire pire, les mêmes règles répétées encore et encore.
